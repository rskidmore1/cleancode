# clean PRs
A check list for making a pull request.


--------------------
**How to start a new ticket:**
1. Read requirements and all documents 2 times
2. Look at functionality in product
3. Read all related code up and down 2 times
  - Pause on tech line for a monent wiht cursor and run eyes over it
4. Comment on where things work and there thing should go
  - Use "Comment: here: " for initial notes and research items
  - Use "todo: here:" for future todos 


---------------------------------
**Finsihing ticket:**
1. Clean up in general
2. Make a draft PR and compare all new code
3. Comment next to every line about it or what it does.
4. Make commit and compare every line to clean up

-----------------------------------------
**Approving and Review PR:**
1. Read all lines up and down 2 times
